# 日常使用指南

恭喜你成功部署了简儿！🎉 这篇文档将帮助你了解简儿的全部功能，以及如何在 QQ 群里使用它们。

## 基本使用方式

简儿在群里的响应有两种触发方式：

### 1. 使用触发符号

在群聊中发送以 **触发符号** 开头的消息，机器人就会响应。触发符号是你在 `config.json` 中设置的 `reminder` 值，默认为 `~`。

例如，如果你的触发符号是 `~`，你可以发送：

```
~帮助
```

机器人就会回复所有可用功能的帮助信息。

### 2. @机器人

在群聊中直接 **@你的机器人**，也可以触发帮助和 AI 对话。

## 功能大全

### 🤖 AI 对话

简儿支持三种 AI 模型：**DeepSeek**、**Google Gemini** 和 **ChatGPT**。启用 AI 功能后，你可以在群里与 AI 进行自然语言对话。

**使用方式**：直接 @机器人 加上你想说的话

**AI 对话的特点**：

- 每个用户有独立的**上下文记忆**，不同人和机器人的对话互不干扰
- 支持为机器人设置**角色扮演**预设（系统提示词）
- 支持 **EdgeTTS 语音回复**，让机器人"说"出来

**管理 AI 上下文**：
- 管理员可以在群里清空所有用户的上下文
- 可以切换不同的 AI 模型

::: tip
AI 功能需要配置 API Key 才能使用。如果你还没有配置，请参考 [配置 AI 功能](/Configuring-AI-Functions)。
:::

### 🎨 娱乐功能

简儿内置了多种有趣的娱乐功能，全部以**插件**形式提供：

| 功能 | 说明 |
| --- | --- |
| **ACG 生图** | 随机生成二次元图片（壁纸、头像等） |
| **Pixiv 搜图** | 根据你给定的标签，从 Pixiv 搜索并发送图片 |
| **名人名言** | 把群友说过的话生成名人名言风格的图片(文字转图片) |
| **大头照** | 生成群友的大头照图片 |
| **Hitokoto 一言** | 随机发送一句优美的句子 |

这些功能都可以通过发送对应的触发关键词来使用。发送 `~帮助` 可以看到完整的功能列表和使用格式。

### 🛡️ 群管理功能

如果你的机器人在群中拥有管理员权限（QQ群管理员），你就可以通过发送指令让机器人执行群管理操作。

**前提条件**：
- 你的 QQ 号需要在简儿的权限组中（至少是 `Manage_User`）
- 机器人在群里需要是**管理员**

**常用群管功能**：

| 指令示例 | 说明 |
| --- | --- |
| `~禁言 @某人 3600` | 禁言某人 3600 秒（1小时） |
| `~解禁 @某人` | 解除某人的禁言 |
| `~踢出 @某人` | 将某人移出群聊 |
| 回复一条消息发送 `撤回` | 撤回那条消息 |
| `~核验 QQ号` | 查询某个 QQ 号的账户信息 |

::: info 关于权限
不是所有人都能使用群管功能。你需要先被添加到机器人的权限组中。详见 [配置用户组](/Configure-User-Group)。
:::

### ⚙️ 机器人管理

作为机器人的管理员（`Manage_User` 及以上权限），你还可以执行以下操作：

| 指令示例 | 说明 |
| --- | --- |
| `~感知` | 查看机器人的运行状态（运行时长、系统信息等） |
| `~修改 hh:mm 内容` | 修改定时群发消息的时间和内容 |
| `~注销` | 清空所有用户的 AI 上下文 |
| `~重启` | 关闭所有线程和进程，然后重新启动简儿 |
| `~让我访问` | 查看拥有管理权限的用户列表 |

### 🧩 插件管理

简儿的功能全部以插件形式运行，你可以在群里方便地管理插件的加载状态：

| 指令 | 说明 |
| --- | --- |
| `启用插件` | 启用某个被禁用的插件 |
| `禁用插件` | 禁用某个正在运行的插件 |
| `重载插件` | 重新从磁盘加载所有插件（无需重启简儿） |
| `插件视角` | 查看当前所有插件的运行状态和报告 |

::: tip 热重载
使用"重载插件"可以在不重启简儿的情况下加载新添加的插件或更新已有插件的代码。这对于开发和调试非常方便。
:::

## 权限组简介

简儿有三个权限等级，从高到低依次为：

| 权限组 | 能力 | 如何获得 |
| --- | --- | --- |
| **ROOT_User** | 最高权限，拥有所有管理能力，且无法被其他人修改 | 只能在 `config.json` 中的 `Others.ROOT_User` 字段设置 |
| **Super_User** | 可以管理其他用户的权限（添加/删除 `Manage_User` 和 `Super_User`） | 由 `Super_User` 或 `ROOT_User` 通过群指令添加 |
| **Manage_User** | 可以执行群管操作、查看机器人状态、管理插件等 | 由 `Super_User` 或 `ROOT_User` 通过群指令添加 |

想了解更多关于权限组的细节，请参考 [配置用户组](/Configure-User-Group)。

## 常见问题

### 机器人在群里不回复我

1. **检查触发符号**：你是否使用了正确的触发符号？发送 `~帮助` 试试（把 `~` 换成你设置的符号）
2. **检查黑名单**：你的 QQ 号是否被加入了 `black_list`？
3. **检查群静默**：机器人所在的群是否被加入了 `silents` 列表？
4. **确认协议端在线**：检查 NapCatQQ 是否正常运行且已登录

### 机器人回复了但 AI 对话不可用

需要在 `config.json` 中配置对应 AI 模型的 API Key，详见 [配置 AI 功能](/Configuring-AI-Functions)。

### 我想让机器人加入更多的群

直接在 QQ 上把机器人邀请进群（或让群主邀请）即可。机器人会自动在所有加入的群中工作。

### 如何停止机器人

- 在命令行窗口按 `Ctrl + C`
- 或者直接关闭运行 `python main.py` 的终端窗口
